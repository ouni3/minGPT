
## minGPT 简介

![mingpt](mingpt.jpg)

minGPT 是一个使用 PyTorch 重新实现的 [GPT](https://github.com/openai/gpt-2) 模型，包含训练和推理部分。minGPT 旨在简洁、清晰、易于理解和学习，因为目前大多数可用的 GPT 模型实现都比较庞大。GPT 本身并不是一个复杂的模型，这个实现大约只有 300 行代码（参见 [mingpt/model.py](mingpt/model.py)）。它的核心原理是将一系列索引输入到 [Transformer](https://arxiv.org/abs/1706.03762) 中，然后输出序列中下一个索引的概率分布。大部分的复杂性在于为了提高效率，在批处理方面（包括跨样本和序列长度）进行了一些巧妙的设计。

**注意 (2023 年 1 月)**：虽然我可能会继续接受并修改一些细节，但 minGPT 处于半存档状态。有关最新进展，请参阅我的重写版本 [nanoGPT](https://github.com/karpathy/nanoGPT)。基本上，minGPT 被广泛引用（笔记本、博客、课程、书籍等），这使得我不太愿意进行更大的改动来推动代码向前发展。我还想稍微改变一下方向，从只关注教育到一些仍然简单易懂但有实用性的东西（重现中等规模的行业基准，接受一些权衡以提高运行时效率等）。

minGPT 库包含三个文件：[mingpt/model.py](mingpt/model.py) 包含实际的 Transformer 模型定义，[mingpt/bpe.py](mingpt/bpe.py) 包含一个稍微重构的字节对编码器，它可以像 OpenAI 在 GPT 中所做的那样在文本和整数序列之间进行转换，[mingpt/trainer.py](mingpt/trainer.py) 是（独立于 GPT 的）PyTorch 样板代码，用于训练模型。然后在 `projects` 文件夹中有一些使用该库的演示和项目：

- `projects/adder` 从头开始训练一个 GPT 来加数字（灵感来自 GPT-3 论文中的加法部分）
- `projects/chargpt` 在一些输入文本文件上训练一个 GPT 作为字符级语言模型
- `demo.ipynb` 以笔记本格式展示了在简单排序示例中 `GPT` 和 `Trainer` 的最小化使用
- `generate.ipynb` 展示了如何加载预训练的 GPT2 并根据一些提示生成文本

### 使用方法

以下是实例化 GPT-2（1.24 亿参数版本）的方法：

```python
from mingpt.model import GPT
model_config = GPT.get_default_config()
model_config.model_type = 'gpt2'
model_config.vocab_size = 50257 # openai 模型的词汇量
model_config.block_size = 1024  # openai 模型的块大小（即输入上下文长度）
model = GPT(model_config)
```

以下是训练模型的方法：

```python
# 你的 torch.utils.data.Dataset 子类，它生成长度不超过 1024 的示例
# torch LongTensor，其整数范围为 [0,50257)
train_dataset = YourDataset()

from mingpt.trainer import Trainer
train_config = Trainer.get_default_config()
train_config.learning_rate = 5e-4 # 许多可能的选项，请参阅文件
train_config.max_iters = 1000
train_config.batch_size = 32
trainer = Trainer(train_config, model, train_dataset)
trainer.run()
```

有关更具体的示例，请参阅 `demo.ipynb`。

论文 + 一些实现说明：

#### 通过生成式预训练改进语言理解 (GPT-1)

- 我们的模型很大程度上遵循了原始的 Transformer 工作
- 我们训练了一个 12 层的仅解码器 Transformer，带有掩码自注意力头（768 维状态和 12 个注意力头）。对于位置感知前馈网络，我们使用了 3072 维的内部状态。
- Adam 最大学习率为 2.5e-4。（后来 GPT-3 对于这个模型大小使用了 6e-4）
- 学习率衰减：在前 2000 次更新中从零线性增加，并使用余弦调度退火至 0
- 我们在 64 个随机抽样的、连续的 512 个标记的最小批次上训练了 100 个 epoch
- 由于在整个模型中广泛使用了层归一化，因此 N(0, 0.02) 的简单权重初始化就足够了
- 字节对编码 (BPE) 词汇表，包含 40,000 次合并
- 残差、嵌入和注意力 dropout，速率为 0.1，用于正则化。
- (37) 中提出的 L2 正则化的修改版本，对所有非偏差或增益权重 w = 0.01
- 对于激活函数，我们使用了高斯误差线性单元 (GELU)。
- 我们使用了学习到的位置嵌入，而不是原始工作中提出的正弦版本
- 对于微调：我们向分类器添加了 dropout，速率为 0.1。学习率为 6.25e-5，批大小为 32。3 个 epoch。我们使用线性学习率衰减调度，在 0.2% 的训练中进行预热。λ 设置为 0.5。
- GPT-1 模型有 12 层，d_model 为 768，约 1.17 亿个参数 


**架构:**

* **Transformer 基础:** GPT-1 的核心架构是 Transformer 模型的解码器部分，这表明了注意力机制在处理序列数据方面的优势。
* **12 层解码器:**  模型使用了 12 个堆叠的解码器层，允许模型学习复杂的语言模式和层次化表示。
* **掩码自注意力:**  每个注意力头只能关注当前位置之前的词语，确保模型在生成文本时不会“看到未来信息”。
* **768 维状态:**  模型内部使用 768 维的向量来表示词语和句子，提供了丰富的语义空间。
* **12 个注意力头:**  多头注意力机制允许模型从不同的角度关注输入序列，捕捉更全面的语义信息。
* **3072 维前馈网络:**  每个解码器层中的前馈网络进一步增强了模型的非线性表达能力。

**训练:**

* **Adam 优化器:**  GPT-1 使用 Adam 优化器来更新模型参数，这是一种高效且常用的优化算法。
* **学习率策略:**  学习率在前 2000 步线性增长，然后使用余弦退火策略逐渐降低，帮助模型稳定地收敛到最优解。
* **批大小和 Epoch:**  模型在 64 个样本的批次上训练了 100 个 epoch，这意味着模型会多次遍历整个训练数据集。
* **权重初始化:**  由于层归一化的使用，简单的 Gaussian 初始化就足够了。
* **BPE 词汇表:**  使用字节对编码 (BPE) 将文本分割成子词单元，有效地处理了词汇表外单词的问题。
* **正则化技术:**  模型使用了 dropout 和 L2 正则化来防止过拟合，提高泛化能力。
* **GELU 激活函数:**  高斯误差线性单元 (GELU) 作为激活函数，引入了非线性变换。
* **学习到的位置嵌入:**  模型学习了每个位置的嵌入向量，而不是使用固定的正弦位置编码。

**微调:**

* **微调参数:**  微调阶段使用了较小的学习率、批大小和 epoch 数，以避免过拟合。
* **线性学习率衰减:**  微调阶段也使用了学习率衰减策略，帮助模型更好地适应下游任务。

**模型规模:**

* **1.17 亿参数:**  GPT-1 包含约 1.17 亿个参数，这在当时是一个相当大的模型。

**总结:**

GPT-1 的架构和训练策略为后续的 GPT 模型奠定了基础。它展示了 Transformer 模型在语言建模方面的强大能力，并引入了许多重要的技术，如掩码自注意力、BPE 词汇表和学习到的位置嵌入。这些技术在后续的自然语言处理研究中得到了广泛的应用和发展。

##  语言模型是无监督的多任务学习者 (GPT-2)

-  层归一化被移动到每个子块的输入，类似于预激活残差网络
-  在最后一个自注意力块之后添加了一个额外的层归一化。
-  使用了一种改进的初始化方法，该方法考虑了残差路径上随模型深度累积的影响。我们在初始化时将残差层的权重缩放 1/√N，其中 N 是残差层的数量。(奇怪的是，在他们发布的代码中，我只能找到对旧的 0.02 的简单使用......在他们发布的 image-gpt 中，我发现它用于 c_proj，即使这样也只用于 attn，而不是 mlp。嗯。https://github.com/openai/image-gpt/blob/master/src/model.py)
-  词汇量扩大到 50,257
-  将上下文大小从 512 个标记增加到 1024 个标记
-  使用更大的批大小 512
-  GPT-2 使用了 48 层和 d_model 1600（而原始模型是 12 层和 d_model 768）。约 15.42 亿个参数

**架构改进:**

* **层归一化位置调整:**  将层归一化操作从子层的输出端移动到输入端，类似于预激活残差网络。这种调整有助于梯度在深层网络中的传播，稳定训练过程。
* **末尾额外层归一化:**  在最后一个自注意力模块之后添加一个额外的层归一化层，进一步稳定训练，并可能提升模型表达能力。

**初始化策略改进:**

* **考虑残差连接的初始化:**  针对残差连接的结构特点，采用了一种改进的初始化方法。通过将残差路径上权重缩放  1/√N (N 为残差层数)，可以有效缓解梯度消失或爆炸问题，使模型更容易训练。

**训练设置改进:**

* **词汇量扩大:**  将词汇量从 40,000 扩展到 50,257，可以更好地捕捉文本中的词汇多样性。
* **上下文长度增加:**  将最大上下文长度从 512 个标记扩展到 1024 个标记，允许模型处理更长的文本序列，捕捉更长距离的语义依赖关系。
* **更大批大小:**  使用更大的批大小 (512)，可以提高训练效率，并可能提升模型的泛化能力。

**模型规模:**

* **48 层，15.42 亿参数:**  GPT-2 的模型规模显著增加，使用了 48 层 Transformer 解码器，参数量达到 15.42 亿，远超 GPT-1。更大的模型规模通常意味着更强的学习能力。

**你提到的代码差异:**

你观察到 GPT-2 论文中描述的初始化方法与代码实现存在差异，这在研究中并不少见。可能的原因包括：

* **简化实现:**  论文中描述的是一种理论上更优的方法，但在实际代码实现中可能为了简化而采用了一种效果接近的近似方法。
* **实验结果差异:**  作者可能在实验中发现，不同的初始化方法对最终结果影响不大，因此选择了更简单的实现方式。
* **代码更新不及时:**  论文发表后，代码可能经过了更新迭代，而论文中的描述没有及时更新。

**总结:**

GPT-2 通过对模型架构、初始化策略和训练设置进行一系列改进，显著提升了语言模型的性能。这些改进为后续的 GPT 模型发展提供了宝贵经验，也推动了自然语言处理领域的技术进步。 

##  语言模型是少样本学习者 (GPT-3)

-  GPT-3：96 层，96 个头，d_model 为 12,288（1750 亿个参数）。
-  类似 GPT-1：12 层，12 个头，d_model 768（1.25 亿个参数）
-  我们使用与 GPT-2 相同的模型和架构，包括其中描述的改进的初始化、预归一化和可逆标记化
-  我们在 Transformer 的层中使用交替的密集和局部带状稀疏注意力模式，类似于稀疏 Transformer
-  我们始终将前馈层的大小设置为瓶颈层的四倍，dff = 4 ∗ dmodel
-  所有模型都使用 nctx = 2048 个标记的上下文窗口。
-  Adam，其中 β1 = 0.9，β2 = 0.95，eps = 10−8
-  所有模型都使用 0.1 的权重衰减来提供少量的正则化。（注意：我认为 GPT-1 使用的是 0.01，见上文）
-  将梯度的全局范数裁剪为 1.0
-  在前 3.75 亿个标记上进行线性学习率预热。然后使用余弦衰减将学习率降低到其值的 10%，持续 2600 亿个标记。
-  在训练的前 40-120 亿个标记（取决于模型大小）中，将批大小从较小的值（32k 个标记）线性增加到完整值。
-  始终使用完整的 2048 大小的上下文窗口，并使用特殊的文档结束标记分隔符

**GPT-3 模型规模：**

* **巨型模型：** GPT-3 最引人注目的特点是其庞大的规模。它拥有 96 层 Transformer 解码器，12,288 维的模型维度，以及高达 1750 亿个参数。这是 GPT-2 的十倍以上，也是当时最大的语言模型。
* **不同规模的模型：**  除了最大的 1750 亿参数模型，OpenAI 还训练了其他规模的 GPT-3 模型，包括与 GPT-1 相似的 1.25 亿参数模型。这为研究不同模型规模对性能的影响提供了宝贵数据。

**架构和训练细节：**

* **继承改进：** GPT-3 继承了 GPT-2 中引入的架构和训练改进，包括改进的初始化策略、预归一化和可逆标记化。
* **稀疏注意力机制：**  为了处理更长的文本序列，GPT-3 采用了稀疏注意力机制，类似于 Sparse Transformer。这种机制只计算部分词语之间的注意力权重，降低了计算复杂度。
* **瓶颈层设计：**  前馈网络的维度始终设置为瓶颈层的四倍，这是一种常见的 Transformer 模型设计选择。
* **固定上下文窗口：**  所有 GPT-3 模型都使用 2048 个标记的固定上下文窗口，这限制了模型处理超长文本的能力。
* **Adam 优化器：**  继续使用 Adam 优化器，并调整了 β1、β2 和 eps 参数。
* **权重衰减：**  使用 0.1 的权重衰减进行正则化，与 GPT-2 相同，但与 GPT-1 (0.01) 不同。
* **梯度裁剪：**  将梯度的全局范数裁剪为 1.0，防止梯度爆炸。
* **学习率策略：**  采用线性预热和余弦退火策略，并根据模型大小调整预热步数。
* **批大小调整：**  在训练初期，线性增加批大小，帮助模型更快地收敛。
* **文档结束标记：**  使用特殊的文档结束标记来分隔不同的文本片段，即使在同一个训练样本中。

**总结：**

GPT-3 通过其前所未有的模型规模、稀疏注意力机制和其他改进，在各种自然语言处理任务上取得了令人瞩目的成果。它展示了扩大语言模型规模的潜力，并引发了对人工智能伦理和社会影响的广泛讨论。 

##  从像素生成式预训练 (Image GPT)

-  在处理图像时，我们选择恒等排列 πi = i，其中 1 ≤ i ≤ n，也称为光栅顺序。
-  我们通过使用 k 均值（k = 512）对 (R, G, B) 像素值进行聚类来创建我们自己的 9 位调色板。
-  我们最大的模型 iGPT-XL 包含 L = 60 层，并使用 d = 3072 的嵌入大小，总共 68 亿个参数。
-  我们的下一个最大模型 iGPT-L 本质上与 GPT-2 相同，具有 L = 48 层，但包含稍小的嵌入大小 d = 1536（而 GPT-2 为 1600），总共 14 亿个参数。
-  我们使用与 GPT-2 相同的模型代码，除了我们以与稀疏 Transformer (Child et al., 2019) 中相同的方式初始化权重，并将所有生成 logits 的投影初始化为零。
-  我们还训练了 iGPT-M，一个具有 L = 36 和 d = 1024 的 4.55 亿参数模型
-  iGPT-S，一个具有 L = 24 和 d = 512 的 7600 万参数模型（好的，有多少个头？看起来 Github 代码声称是 8 个）
-  在预训练 iGPT-XL 时，我们使用 64 的批大小并训练 200 万次迭代，对于所有其他模型，我们使用 128 的批大小并训练 100 万次迭代。
-  Adam，其中 β1 = 0.9，β2 = 0.95
-  学习率预热一个 epoch，然后衰减到 0
-  我们没有使用权重衰减，因为应用 0.01 的小权重衰减不会改变表示质量。
-  iGPT-S 学习率 0.003
-  未使用 Dropout。


**图像处理：**

* **光栅顺序 (Raster Order):** iGPT 使用光栅顺序处理图像像素，这意味着模型按从左到右、从上到下的顺序逐个处理像素。
* **9 位调色板：**  为了降低计算复杂度，iGPT 使用 k 均值聚类将图像像素颜色量化为 9 位调色板。

**模型规模和架构：**

* **iGPT-XL (最大模型):**  拥有 60 层 Transformer 解码器，嵌入维度为 3072，总共 68 亿个参数。
* **iGPT-L:**  与 GPT-2 架构相似，拥有 48 层，但嵌入维度略小 (1536)，总共 14 亿个参数。
* **iGPT-M:**  36 层，嵌入维度 1024，总共 4.55 亿个参数。
* **iGPT-S (最小模型):**  24 层，嵌入维度 512，总共 7600 万个参数。根据 Github 代码，iGPT-S 使用了 8 个注意力头。

**训练细节：**

* **模型代码:**  iGPT 的模型代码主要基于 GPT-2，但进行了一些调整，例如权重初始化方式。
* **权重初始化:**  iGPT 采用了与稀疏 Transformer 相同的权重初始化方法，并将生成 logits 的投影层初始化为零。
* **批大小和迭代次数:**  iGPT-XL 使用 64 的批大小，训练 200 万次迭代；其他模型使用 128 的批大小，训练 100 万次迭代。
* **优化器:**  使用 Adam 优化器，β1 = 0.9，β2 = 0.95。
* **学习率策略:**  学习率在一个 epoch 内预热，然后衰减到 0。
* **权重衰减:**  iGPT 没有使用权重衰减，因为实验表明它对表示质量没有显著影响。
* **Dropout:**  iGPT 没有使用 Dropout 正则化技术。

**总结：**

iGPT 将 Transformer 模型成功应用于图像生成领域，证明了 Transformer 在处理不同类型数据方面的强大能力。iGPT 的不同模型规模也为研究模型规模对图像生成质量的影响提供了 valuable insights。
